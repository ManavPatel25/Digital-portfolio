04. Interdisciplinary Projects & Research

Childhood Obesity Risk Analysis (Research)

An interdisciplinary research project blending public health with data science to identify predictive factors for childhood obesity.

  • Objectives: To build a predictive model using machine learning to identify key risk factors for childhood obesity.
  • Methodologies: Utilized Python with Pandas, Matplotlib, and Scikit-learn on a validated dataset from the UCI Machine Learning Repository.
  • Outcomes & Innovations: The research paper identified Gradient Boosting and Random Forest models as top performers, achieving over 93% accuracy in predicting obesity risk.
  • Challenges & Solutions: The primary challenge was handling a noisy and incomplete dataset. This was solved by implementing robust data imputation techniques and feature engineering to derive meaningful insights.

Customer Support Ticket Management System

A full-stack web application designed to streamline customer support operations for a small business.

  • Objectives: To create a platform where users can submit support tickets, and admins can track, assign, and resolve them.
  • Methodologies: Built using PHP for the backend logic, MySQL for the database, and HTML/CSS/JavaScript for a dynamic frontend. Implemented user authentication and role-based (user/admin) access control.
  • Outcomes & Innovations: A complete, deployable web application with a secure login system, intuitive admin dashboard (as shown in the 'Skills' section), and a simple ticket submission form.
  • Challenges & Solutions: Ensuring data integrity and preventing SQL injection was a key challenge, addressed by using prepared statements and sanitizing all user inputs.

Digital Electronics Heat Detector

A foundational hardware project integrating principles of digital electronics and sensor technology.

  • Objectives: To design and build a circuit that activates an alarm (LED/buzzer) when the ambient temperature crosses a predefined threshold.
  • Methodologies: Employed logic gates (AND, NOT), a thermistor as the temperature sensor, and an op-amp as a comparator to create a simple, reliable warning system.
  • Outcomes & Innovations: A functional hardware prototype that successfully demonstrated the practical application of digital logic principles.
  • Challenges & Solutions: Calibrating the thermistor's resistance to the op-amp's threshold voltage was tricky. This was solved through iterative testing and using a potentiometer for fine-tuning.

Artefact: Admin Dashboard Logic (PHP)

This snippet from the 'Customer Support' project's admin dashboard (`dashboard.php`) demonstrates session management, complex SQL queries to count ticket statuses, and logic for pagination.

// Helper to count tickets
function countTickets($conn, $where = '') {
    $query = "SELECT COUNT(*) AS total FROM tickets";
    if ($where) $query .= " WHERE $where";
    $res = mysqli_query($conn, $query);
    if (!$res) return 0;
    $row = mysqli_fetch_assoc($res);
    return (int)$row['total'];
}

// Ticket counters
$totalTickets   = countTickets($conn);
$openTickets    = countTickets($conn, "status='Open'");
$pendingTickets = countTickets($conn, "status='Pending'");

// Pagination setup
$limit = 10;
$page = isset($_GET['page']) && is_numeric($_GET['page']) ? (int)$_GET['page'] : 1;
$offset = ($page - 1) * $limit;

// Fetch tickets for this page
$result = mysqli_query($conn, "SELECT * FROM tickets ORDER BY created_at DESC LIMIT $limit OFFSET $offset");

07. Global Awareness & Ethical Considerations

Case Study: Ethical Data Handling in Obesity Research

My research project on childhood obesity was not just a technical challenge; it was an ethical one. The dataset contained sensitive, non-public information about minors.

Analysis of Ethical Implications: The primary ethical dilemma was the "dual-use" nature of data. While our goal was positive (public health insight), the same data, if leaked or misused, could lead to stigmatization, discrimination, or breaches of privacy for the individuals involved. This is a microcosm of a global issue facing tech giants and health organizations daily: the conflict between data utility and individual privacy.

Ethical Framework Applied: We operated under a strict "Privacy by Design" framework.

  • Anonymization: All personally identifiable information (PII) was removed from the dataset before we ever received it.
  • Data Minimization: We only analyzed the specific features necessary for the research question, ignoring other potentially sensitive columns.
  • Secure Storage: The data was stored in an encrypted environment and was never uploaded to a public GitHub repository.

Personal Reflections: This project profoundly shaped my view of a developer's responsibility. It's not enough to just write code that works. We must be the frontline guardians of data privacy. It taught me to always ask "Should we build this?" and "What are the risks?"—not just "Can we build this?". This ethical-first mindset is a non-negotiable part of my professional identity.
